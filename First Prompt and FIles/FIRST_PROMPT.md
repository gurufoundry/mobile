# First Prompt for Claude Code

Copy everything below the line into your first Claude Code session.

---

I'm starting a new project called **Sonic Sticker** — a web app where users chat with an AI agent to design custom stickers, attach voice notes, and export print-ready files paired with NFC tags.

I have a complete spec in `SPEC.md` in this folder. Please:

1. **Read `SPEC.md` end-to-end before doing anything else.** Don't skim. The spec has tech decisions, scope boundaries, build order, and constraints I want enforced.

2. After reading, **summarize back to me** in 5–8 bullets: what we're building, the stack, the MVP scope (in vs. out), and what Phase 0 will produce. This is so I can confirm you understood it before we touch any code.

3. Then **ask me the confirmation questions** listed in section 9 of the spec ("Things to confirm before starting"). Don't proceed without my answers.

4. Once I've answered, we'll start **Phase 0 (Skeleton) only**. Do not jump ahead to later phases. At the end of Phase 0, stop and wait for me to test before moving on.

A few standing rules for our whole collaboration:

- The constraints in section 10 of the spec are non-negotiable. Bring them up if I ever ask you to violate one.
- If I ask for a feature that's in the "out of scope" list (section 3), push back and remind me.
- For any decision the spec doesn't cover, ask before assuming.
- Keep commits small and meaningful. Suggest commit messages, don't auto-commit silently.
- When you're about to use a library, package, or pattern I might not know, give me a one-sentence "why this" before installing.

Ready when you are. Start with the spec.
